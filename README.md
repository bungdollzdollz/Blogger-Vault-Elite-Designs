## ✦ What Makes These Templates Different?

### 1. Modular Component Architecture
Each template is built from independent blocks: header, hero, content grid, footer. You can mix components from different templates without breaking the layout. Think of it as LEGO bricks for your blog’s visual identity.

### 2. Adaptive Typography System
- Fluid type scaling from 320px to 2560px viewports  
- Variable font support with fallback stacks  
- Readability-tested line heights (1.5 for body, 1.2 for headings)

### 3. Asset-Light Philosophy
The entire template footprint—including CSS, JavaScript, and critical inline styles—averages **under 18KB**. This is achieved through:
- Tree-shaken utility classes  
- Lazily loaded non-critical assets  
- No external font dependencies (system font stack optimized)

### 4. SEO Architecture
- Schema.org markup for articles, breadcrumbs, and sitelinks  
- Proper header hierarchy (H1 → H6)  
- Open Graph and Twitter Card meta templates  
- Canonical URL handling via data attributes

### 5. Global Readiness
- RTL layout support prepared in the base grid  
- Date formatting adaptable to regional conventions  
- Unicod-verified for scripts like Devanagari, Arabic, and CJK

## ✦ How to Use These Templates

1. **Browse** the `/templates` directory to preview the collection. Each template has its own folder with a `preview.html` file.
2. **Identify** your requirements: Do you need a sidebar? Two-column footer? Author bio section?
3. **Select** the template JSON manifest that matches your needs. The manifest contains all styling variables and widget placements.
4. **Apply** the template to your Blogger instance via the **Theme → Backup/Restore → Upload** workflow.
5. **Customize** using the included `_variables.xml` file—change colors, fonts, and spacing without touching the core CSS.

> **Note:** The templates are **MIT-licensed**, meaning you can modify, redistribute, or use commercially without attribution. However, attribution is appreciated as a nod to the community effort.

## ✦ Code Philosophy

> "A template should be invisible—the reader should see content, not columns."

This repository avoids:
- Overly nested divs (maximum depth of 4)
- Inline styles (except critical above-the-fold content)
- Unnecessary animations that drain battery or bandwidth
- Vendor-specific polyfills that bloat the payload

Every line of code is reviewed for **performance impact** before merging. The median template adds only **12 HTTP requests** to your page load.
